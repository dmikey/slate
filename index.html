    Introduction

    Welcome to Chemical. Chemical UI Framework lets you take as little, or as much as you want.

    Chemical provides CSS if all you want is CSS styles to get moving. We also provide Stylus files, to preprocess with your project.

    For those who want more control, we provide isomorphic vanilla javascript objects that help you draw server side views, and quickly paint those same views to the screen on the client.

    Chemical providers a number of UI widgets, drawing components, a client side router, and ajax. There is no data binding here, or cruft. Emphasis is on building ultra fast and beautiful applications.

    Include Chemical In your Project to get started

    npm install chemical --save

    npm install chemical --save
    If you are working with Chemical in JavaScript, we recommend you use webpack or browserify to build your application
    You can use Require1k.js to work with your application code unbuilt in the browser!

    Style Only

    We get it. Sometimes you just want to create a beautiful interface, but on your terms. Maybe you don’t want our resets/normalizers, maybe you do.

    We provide a built and minified version of our CSS Framework. You can use these in your pages to start right away. They come with all of our widgets out of the box! You can find them in HTML Samples tab!

    You also have access to our Stylus files. So if you want to can bring along exactly the styles you use for your application!

    Include Chemical In your Project

    
<link rel="stylesheet" type="text/css" href="./node_modules/chemical/chemical.css"/>


    // For Webpack
//
// install an appropriate loader for stylus
// require the node module, and the widget style

// require only a single widget
require('chemical/button.styl');

// or load everything
require('chemical/chemical.styl');

    The Basics

    Chemical tries to help alleviate some of the strain when building a UI. We provide a few out of the box components, that will help you draw a UI.

    Chemical recommends one way data binding, being aware of your application state, and tries to provide minimal UI plumbing to ensure you can develop an ultra fast, and ultra light UI.

    Chemical recommends you use browserify or webpack to bundle your application. This way, your app only brings the exact parts of Chemical along, that it needs.

    When creating new widgets or components, Chemical has chosen to only be of the opinion that we require a templating function that is passed an object literal to use for filling data into the template. Later on you can learn some of our techniques for creating seamless HTML to ultra fast compiled JS templates.

    href: '#'
